# Kagi Sidebar

Kagi Sidebar is an unofficial CSS-only customization for Kagi Custom CSS. It
moves the standard web search filters into a compact left sidebar on desktop for
both JS-enhanced and basic/no-JS web search, while preserving Kagi's native
search behavior, links, forms, dropdowns, and result rendering.

## Kagi ustom CSS Previewer

This project also includes a previewer to locally test Kagi custom CSS. It may be easier than repeatedly updating from Kagi settings. Especially for coding agents.

## Proposal: Semantic Hooks and Components for Kagi Custom CSS

As I wrote the custom CSS to convert the search filters into a sidebar, I thought of ways Kagi could make it easier for authors of custom CSS. So I documented my thoughts. Providing a way to add custom CSS is nice. Providing convenient CSS selectors so that custom CSS is easier to write and maintain would be even nicer!

Read the proposal: https://github.com/Leftium/kagi-sidebar/blob/main/proposal/kagi-semantic-hooks-and-components.md 


## Kagi CSS/HTML optimization experiment

The custom CSS proposal got me wondering if it would be possible to optimize the HTML/CSS generated by Kagi. I think there is definitely room for improvement, but I'm not sure if it would be worth the effort. Especially when working from the "outside," with only access to the final HTML/CSS generated by Kagi. I moved the experiment to a separate branch: https://github.com/Leftium/kagi-sidebar/tree/kagi-css-html-optimization-experiment

## Commands

Install dependencies with pnpm before running the previewer:

```sh
pnpm install
```

Useful commands:

```sh
pnpm generate # prepare generated preview pages
pnpm dev      # open the Vite previewer
pnpm check    # run cheap JavaScript syntax checks
pnpm format   # format tracked source files
```

`generated/` is disposable. Regenerate it when captures, Custom CSS files, or
previewer tools change.

## Main Files

- `src/kagi-sidebar.css` is the pasteable Kagi Custom CSS stylesheet.
- `specs/kagi-sidebar.md` documents the sidebar release scope and behavior.
- `previewer/` opens captured Kagi pages with `src/kagi-sidebar.css` or extra
  local Custom CSS files.
- `proposal/` keeps the maintainer-facing semantic hooks and components
  proposal for reference.
